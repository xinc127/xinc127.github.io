创建新的文章

根目录下命令行输入hexo new <模板> <文章名> 新建文章

其中的模板在scaffolds文件夹中，默认有三种模板post、draft、page

参数 功能 路径
post 新建文章 /source/_posts
draft 新建草稿 /source/_draft
page 新建页面（标签页、分类页等） /source/

通常使用hexo new post <title>新建文章，这里的<titel>不是文章名字。

例如新建一篇文章 hexo new post ariticles

然后在source/_posts文件夹下面多了一个articles.md文件

打开文件后（使用的Typora编辑器打开）

在其中配置文章相关内容，标签、分类等。其中title为文章发表后的文章页面显示的标题。

而.md文件的名字，将作为文章的链接地址显示，例如本篇的文件名：articles.md，那么本篇文章的地址为http://xxx.xxx/ariticles.html

文章模板

在新建文章的post参数就是模板的名称，在使用hexo new post <title>命令新建文章时，系统会自动在scaffolds文件夹下寻找post文件，根据该配置文件生新的.md文件

概念: 设备的pixels和CSS的pixels

设备pixels大多数情况下可以通过screen.width/height上获取到。

当用户缩放浏览器时就是将CSS的pixels变大，缩放200%即将1个CSS的pixels变成了4个设备的piexels的大小，即宽度 2、高度 2，面积扩大了2 * 2.

用户缩放时，浏览器自动使CSS的pixels缩放

100% 缩放

在缩放级别为100%时，1单位的CSS的pixels是严格相等于1单位的设备pixels

屏幕尺寸 Screen size

screen.width/height

  • 用户屏幕的完整大小

  • 显示器特性与浏览器无关

浏览器尺寸 Window size

window.innerWidth/Height

  • 包含滚动条尺寸浏览器的完整大小

滚动移位 Scrolling offset

window.pageX/YOffset

  • 页面的位移

window.pageXOffset 和 window.pageYOffset，定义了页面（document）的相对于窗口原点的水平和垂直位移。因此可以知道用户将页面滚动了多少。

缩放不影响window.pageX/YOffset的大小

视窗 viewport

决定了页面中最高块状（block）元素： 元素 的大小

宽度受 viewport 所限制， 元素宽度为 viewport 的100%.

通常，一个块级元素占其父级元素的100%宽度。所以的宽度就是其父元素的100%.

  • viewport宽度等于浏览器宽度
  • 缩放比例100%显示正常，放大浏览器时，viewport变得比网站的宽度更小，对viewport无影响，但页面内容溢出了元素

页面的宽度 document.width

度量 viewport

doucment.documentElement.clientWidth/Height

  • viewport 的尺寸

document.documentElement 就是 元素：HTML文档的根元素

虽然document.documentElement好像是从元素获取值，但实际上是viewport.通过给元素赋值，再通过docuent.documentElement获取到的值不变。（不建议给元素赋值）

这个特殊的规则只针对这个特殊的情况下，在其余情况下等于元素使用的实际宽度。

两个特性对 two property pairs

用来描述viewport尺寸的还有 window.innerWidth/Height

两者的区别：

  • window.innerWidth/Height 包含滚动条
  • document.documentElement.clientWidth/Height 不包含

度量 元素

document.documentElement.offsetWidth/Height

用来获取元素的尺寸

事件坐标 Event coordinates

pageX/Y、clientX/Y、screenX/Y分别是

  • 从原点到事件触发的CSS的pixels
  • 从浏览器（viewport）原点到事件触发的CSS的pixels
  • 从显示器原点到事件触发的CSS的pixels
