浅谈 URI、URL、src、href

URI 和 URL

与 URI(统一资源标识符) 相比，我们应该比较熟悉 URL (统一资源定位符)。

URL 正是我们使用浏览器时候输入的网页地址，比如，下面 http://www.google.com 就是 URL。

URI 统一资源标识符

URI 是 Uniform Resource Identifier 的缩写。

  • Uniform

    规定统一的格式方便可方便处理多种不同类型的资源。比如常用的 HTTP HTTPS 协议，除此之外，还有 ftp、mailto、telnet、file 等。

  • Resource

    资源的定义是“可以标识的任何东西”。不仅仅是文档文件、图片或者其他服务等能够区别于其他类型的，全都可以作为资源。另外，资源不仅可以是单一的，也可以是多数的集合体。

  • Identifier

    表示可标识的对象，也称为标识符。

综上， URI 就是由某个协议方案表示的资源的定位标识符。该协议方案就是访问资源所使用的协议类型。

而 URL 表示资源的地点(互联网所处的位置)，可以将其理解为是一种 URI 。

URI 格式

表示指定的 URI ，要使用涵盖全部信息的绝对 URI 、绝对 URL 以及相对 URL 。相对 URL ，是指从浏览器中基本 URI 处指定的 URL ，比如 /assets/logo.png 。

href 和 src

两者通常都是用来外部资源的引入。

它们的区别在于，src 用来代替标签元素的内容，而 href 是建立标签元素和外部资源的联系。
