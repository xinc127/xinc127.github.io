正规表示法与文件格式化处理 grep

1、什么是正规表示法

简单说，正规表示法就是处理字符串的方法，他是以行为单位进行字符串的处理行为，正规表示法透过一些特殊符号的辅助，可以让使用者轻易达到【搜寻/删除/取代】某特定子串的处理程序！

正规表示法基本是一种【表示法】，只要工具支持这种表示法，那么该工具就可以用正规表示法来处理字符串。例如 vi、grep、awk、sed 等。但例如 cp、ls 等指令就不支持正规表示法。

Polyfill

Polyfill或者Polyfiller，用来形容“用javascript来实现一些浏览器不支持的原生API”。

Shim一般有自己的API，而不只是单纯实现原生不支持的API。(即Shim包含Polyshill)

Polyfill的准确意思是：用于实现浏览器不支持的原生API的代码。

例如，jQuery不是一个Polyshill。它不是实现标准的API，而是封装了自己的API。

一个Polyshill是对旧浏览器实现新标准原生API的封装。

Module 的语法

ES6之前的模块解决方案有CommonJS和AMD两种。前者用于服务器，后者用于浏览器。

ES6模块的设计思想是尽量静态化，在编译时就能确定模块之前的依赖关系，以及输入输出变量。CommonJS和AMD模块，都只能在运行时确定这些东西。比如，CommonJS模块就是对象，输入时必须查找对象的属性。

ES6模块不是对象，而是通过export显式地指定输出代码，再用import命令输入。

严格模式

ES6模块自动采用严格模式，不管你有没有在头部添加”use strict“；。

严格模式主要有以下限制。

  • 变量必须声明后再使用
  • 函数的参数不能有同名属性，否则报错
  • 不能使用with语句
  • 不能对只读属性赋值，否则报错
  • 不能使用前缀 0 表示八进制数，否则报错
  • 不能删除不可删除的属性，否则报错
  • 不能删除变量delete prop，会报错，只能删除属性delete global[prop]
  • eval不会在它的外层作用域引入变量
  • eval和arguments不能被重新赋值
  • arguments不会自动反映函数参数的变化
  • 不能使用arguments.callee
  • 不能使用arguments.caller
  • 禁止this指向全局对象
  • 不能使用fn.caller和fn.arguments获取函数调用的堆栈
  • 增加了保留字（比如protected、static和interface）

注，ES6模块中，顶层的this指向undefined，即不可在顶层代码中使用this。

export 命令

模块功能的两个主要命令export和import。export用于指定模块对外输出的接口，import用于输入其他模块的内容。

模块是一个独立的文件，外部无法获得其内部的变量，只能通过export输出变量。

export还可以输出函数和类

注意，export命令规定的是对外的接口，必须和与模块内部的变量建立一一对应的关系。

上面代码规定了对外的接口m，其他脚本可通过这个接口，取得值1；实质上是在接口名与模块内部变量建立了一一对应的关系。

另外，export输出的是接口，与其对应的值是动态绑定的关系，即通过该接口，可以获取到模块内部实时的值。

最后，export命令必须放在模块代码的在外层代码中的任何位置，不可以在其他块级作用域中。这是因为模块是在编译过程中静态加载的，如果放在条件语句中将会报错。

import 命令

使用export命令定义了模块对外的接口以后，其他js文件就可以了通过import加载该模块了。

上面代码使用import命令加载profile.js模块，并从中进行输入变量。import命令接受一对大括号，里面指定从其他模块导入的变量。导入变量名必须与输出的变量名相同。

同样，import命令可是使用as关键词对引入的变量进行重命名。

import命令输入的变量都是只读的，因为其本质上是输入接口。

import后面的from指定了输入模块的位置，可以是相对路径也可以是绝对路径，.js后缀可以省略。如果只是模块名，那么需要有指定的配置文件，说明该模块的位置。

注意，import有变量提升的效果，因此import命令可以放在代码的任意位置。同export命令，不可以放在语句当中。

多次执行import语句，只会执行一次。

export default 命令

import命令需要指定加载的变量名或者函数名。

当模块输出使用export default命令，使用import命令加载时可以指定任意名字。

注意，import默认输出export default时不用加大括号。

事实上，export default输出的是defaule变量，所以它的后面不能再跟变量声明语句了。

创建新的文章

根目录下命令行输入hexo new <模板> <文章名> 新建文章

其中的模板在scaffolds文件夹中，默认有三种模板post、draft、page

参数 功能 路径
post 新建文章 /source/_posts
draft 新建草稿 /source/_draft
page 新建页面（标签页、分类页等） /source/

通常使用hexo new post <title>新建文章，这里的<titel>不是文章名字。

例如新建一篇文章 hexo new post ariticles

然后在source/_posts文件夹下面多了一个articles.md文件

打开文件后（使用的Typora编辑器打开）

在其中配置文章相关内容，标签、分类等。其中title为文章发表后的文章页面显示的标题。

而.md文件的名字，将作为文章的链接地址显示，例如本篇的文件名：articles.md，那么本篇文章的地址为http://xxx.xxx/ariticles.html

文章模板

在新建文章的post参数就是模板的名称，在使用hexo new post <title>命令新建文章时，系统会自动在scaffolds文件夹下寻找post文件，根据该配置文件生新的.md文件
