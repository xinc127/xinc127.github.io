TypeScript 是什么

对于不熟悉 TypeScript 的人来说，这是一种在 JavaScript 之上添加类型语法的语言，可以通过称为静态类型检查的过程对其进行分析。这种类型检查可以在我们运行代码之前进行告诉我们有关错误（例如错别字和值）的潜在信息null，undefined。不仅如此，类型分析还可以为 TypeScript 和 JavaScript 提供可靠的编辑体验，并为诸如代码完成，全部引用，快速修复和重构之类的操作提供强大支持。

TypeScript is a typed superset of JavaScript that compiles to plain JavaScript.

TypeScript 是一个编译到纯 JavaScript 的有类型定义的JS超集。

TS遵循当前以及未来出现的ECMAScript规范。TS不仅能兼容现有的JavaScript 代码，它也拥有兼容未来版本的JavaScript的能力。大多数TS的新增特性 都是基于未来的JavaScript提案，这意味着许多TS代码在将来很有可能会变成ECMA的标准。

4. class 类

传统方法中，JavaScript 通过构造函数实现类的概念，通过原型链实现继承。而在 ES6 中，我们终于迎来了 class（语法糖）。TypeScript 除了实现了所有 ES6 中的类的功能以外，还添加了一些新的用法。

4.1. 类的概念

简单的介绍：

  • 类(Class)：定义了一件事物的抽象特点，包含它的属性和方法
  • 对象（Object）：类的实例，通过 new 生成
  • 面向对象（OOP）的三大特性：封装、继承、多态
  • 封装（Encapsulation）：将对数据的操作细节隐藏起来，只暴露对外的接口。外界调用端不需要（也不可能）知道细节，就能通过对外提供的接口来访问该对象，同时也保证了外界无法任意更改对象内部的数据
  • 继承（Inheritance）：子类继承父类，子类除了拥有父类的所有特性外，还有一些更具体的特性
  • 多态（Polymorphism）：由继承而产生了相关的不同的类，对同一个方法可以有不同的响应。比如 Cat 和 Dog 都继承自 Animal，但是分别实现了自己的 eat 方法。此时针对某一个实例，我们无需了解它是 Cat 还是 Dog，就可以直接调用 eat 方法，程序会自动判断出来应该如何执行 eat
  • 存取器（getter & setter）：用以改变属性的读取和赋值行为
  • 修饰符（Modifiers）：修饰符是一些关键字，用于限定成员或类型的性质。比如 public 表示公有属性或方法
  • 抽象类（Abstract Class）：抽象类是供其他类继承的基类，抽象类不允许被实例化。抽象类中的抽象方法必须在子类中被实现
  • 接口（Interfaces）：不同类之间公有的属性或方法，可以抽象成一个接口。接口可以被类实现（implements）。一个类只能继承自另一个类，但是可以实现多个接口

URI 和 URL

与 URI(统一资源标识符) 相比，我们应该比较熟悉 URL (统一资源定位符)。

URL 正是我们使用浏览器时候输入的网页地址，比如，下面 http://www.google.com 就是 URL。

URI 统一资源标识符

URI 是 Uniform Resource Identifier 的缩写。

  • Uniform

    规定统一的格式方便可方便处理多种不同类型的资源。比如常用的 HTTP HTTPS 协议，除此之外，还有 ftp、mailto、telnet、file 等。

  • Resource

    资源的定义是“可以标识的任何东西”。不仅仅是文档文件、图片或者其他服务等能够区别于其他类型的，全都可以作为资源。另外，资源不仅可以是单一的，也可以是多数的集合体。

  • Identifier

    表示可标识的对象，也称为标识符。

综上， URI 就是由某个协议方案表示的资源的定位标识符。该协议方案就是访问资源所使用的协议类型。

而 URL 表示资源的地点(互联网所处的位置)，可以将其理解为是一种 URI 。

URI 格式

表示指定的 URI ，要使用涵盖全部信息的绝对 URI 、绝对 URL 以及相对 URL 。相对 URL ，是指从浏览器中基本 URI 处指定的 URL ，比如 /assets/logo.png 。

href 和 src

两者通常都是用来外部资源的引入。

它们的区别在于，src 用来代替标签元素的内容，而 href 是建立标签元素和外部资源的联系。

Module 的语法

ES6之前的模块解决方案有CommonJS和AMD两种。前者用于服务器，后者用于浏览器。

ES6模块的设计思想是尽量静态化，在编译时就能确定模块之前的依赖关系，以及输入输出变量。CommonJS和AMD模块，都只能在运行时确定这些东西。比如，CommonJS模块就是对象，输入时必须查找对象的属性。

ES6模块不是对象，而是通过export显式地指定输出代码，再用import命令输入。

严格模式

ES6模块自动采用严格模式，不管你有没有在头部添加”use strict“；。

严格模式主要有以下限制。

  • 变量必须声明后再使用
  • 函数的参数不能有同名属性，否则报错
  • 不能使用with语句
  • 不能对只读属性赋值，否则报错
  • 不能使用前缀 0 表示八进制数，否则报错
  • 不能删除不可删除的属性，否则报错
  • 不能删除变量delete prop，会报错，只能删除属性delete global[prop]
  • eval不会在它的外层作用域引入变量
  • eval和arguments不能被重新赋值
  • arguments不会自动反映函数参数的变化
  • 不能使用arguments.callee
  • 不能使用arguments.caller
  • 禁止this指向全局对象
  • 不能使用fn.caller和fn.arguments获取函数调用的堆栈
  • 增加了保留字（比如protected、static和interface）

注，ES6模块中，顶层的this指向undefined，即不可在顶层代码中使用this。

export 命令

模块功能的两个主要命令export和import。export用于指定模块对外输出的接口，import用于输入其他模块的内容。

模块是一个独立的文件，外部无法获得其内部的变量，只能通过export输出变量。

import后面的from指定了输入模块的位置，可以是相对路径也可以是绝对路径，.js后缀可以省略。如果只是模块名，那么需要有指定的配置文件，说明该模块的位置。

注意，import有变量提升的效果，因此import命令可以放在代码的任意位置。同export命令，不可以放在语句当中。

多次执行import语句，只会执行一次。

模块整体加载

除了指定加载某些值以外，还可以使用整体加载，即用*指定一个对象，所有的输出值都加载在这个对象身上。
